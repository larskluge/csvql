# AGENTS.md

This file provides guidance to agents when working with code in this repository.

## Build Commands

```bash
make generate    # Run xcodegen to regenerate csvql.xcodeproj (REQUIRED before any build)
make test        # Build and run all unit tests
make install     # Release build + copy to /Applications + QuickLook registration
make clean       # Clean build artifacts
```

Run a single test class:
```bash
xcodegen generate && xcodebuild -project csvql.xcodeproj -scheme csvqlTests \
  -destination 'platform=macOS' \
  -only-testing:csvqlTests/CSVParserTests test
```

After `make install`, test QuickLook via Finder space bar (NOT `qlmanage -p` — see Gotchas).

## Architecture

macOS document-based app with an embedded QuickLook extension. Four xcodegen targets defined in `project.yml`:

- **csvql** — Host app (NSDocument + WKWebView, interactive: sort/filter/select/density)
- **csvqlPreview** — QuickLook extension (QLPreviewingController + WKWebView, static HTML)
- **csvqlTests** — Unit tests (XCTest)
- **csvql-screenshot** — CLI for generating screenshots

All CSV processing lives in `Shared/` and compiles into both app and extension targets.

### Data Pipeline

```
File bytes → DelimiterDetector → CSVParser → TypeInferrer → CSVData → CSVRenderer → HTML
```

- **DelimiterDetector** — Scores tab/comma/semicolon/pipe by consistency across first 10 lines
- **CSVParser** — State-machine parser using Unicode scalar iteration (not Character — Swift merges \r\n into one grapheme)
- **TypeInferrer** — Per-column inference with 80% threshold. Priority: bool → number → date → link → email → sha → text
- **CSVRenderer** — Generates self-contained HTML. Static mode for QuickLook, interactive mode (JS for sort/filter/select) for host app

### Host App UI

- `AppDelegate.swift` — `@main enum Main` pattern for activation policy setup
- `MainMenu.swift` — Programmatic menu: View menu has density toggles (Compact ⌘1, Regular ⌘2, Comfortable ⌘3)
- `DocumentController.swift` — NSDocument subclass, loads CSV via the shared pipeline
- `DocumentWindowController.swift` — WKWebView container with JS bridge for density/sort/filter

### Signing & Registration

`scripts/install.sh` handles ad-hoc codesigning (no `--deep` — preserves extension signature), lsregister cleanup, pluginkit registration. The app must launch briefly for macOS to discover the QuickLook extension.

## QuickLook Limitations

- **CSV QuickLook is blocked by Apple's `Office.qlgenerator`** (`/System/Library/QuickLook/Office.qlgenerator`) which claims `public.comma-separated-values-text` at priority 48. System `.qlgenerator` plugins unconditionally win over third-party `.appex` extensions for the same UTI. No plist key, API, or priority setting can override this. SIP protects the system generator from modification. Commercial apps (Peek) and open-source projects (SourceCodeSyntaxHighlight) confirm this as an unsolvable Apple limitation.
- **TSV QuickLook works** because no system generator claims `public.tab-separated-values-text`.
- **`qlmanage -p` does NOT test `.appex` extensions** — it only invokes legacy `.qlgenerator` plugins. Always test QuickLook via Finder space bar.
- **QLPreviewProvider does not help** — switching from `QLPreviewingController` to `QLPreviewProvider` with `QLIsDataBasedPreview=true` does not change dispatch priority; it only changes how data is delivered to the extension.

## Info.plist Structure

- **`NSDocumentClass` must be INSIDE the document type dict** in `CFBundleDocumentTypes`, not at the plist root level. Placing it at root causes "cannot open files in the X format" errors on double-click.
- **`QLSupportedContentTypes` and `QLIsDataBasedPreview` must be nested inside `NSExtension > NSExtensionAttributes`** in the QuickLook extension's Info.plist, not at the plist root level. Incorrect nesting silently prevents the extension from being dispatched.
- Both plists are hand-edited (`GENERATE_INFOPLIST_FILE: false` in project.yml), not generated by xcodegen.

## Gotchas

- Always run `xcodegen generate` before building — the .xcodeproj is generated, not checked in
- SourceKit "No such module XCTest" warnings are IDE noise; xcodebuild passes fine
- Dark theme only — no light mode support
- No file watching — files are read once on load
- No XPC service (unlike the reference project mdql)
- Test fixtures in `csvqlTests/Fixtures/` must use anonymized data only
- `Shared/Resources` must use `buildPhase: resources` under `sources` in project.yml (not a separate `resources` key) for xcodegen to include CSS in bundles
- CSVParser uses Unicode.Scalar iteration (not Character) because Swift merges \r\n into a single grapheme cluster
